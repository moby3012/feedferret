# v1.3 — Feature Backlog Release

> Status: Planned
> Estimated effort: M–L (depends on scope selection at planning time)
> Depends on: v1.2

---

## Goal & Scope

v1.3 draws from the defined backlog to ship a set of high-demand features that did not fit the themes of v1.1 or v1.2. Exact scope is confirmed at the start of the release cycle based on user demand and engineering capacity. Candidate features are listed below with effort estimates.

---

## Candidate Features

### Fever API Compatibility ✅ Shipped

The Fever API protocol is implemented in `app/api/fever/route.ts` (`?api` handshake, `?feeds`, `?feeds_groups`, `?items`, `?unread_item_ids`, `?saved_item_ids`, `mark=item/feed`), authenticated by `md5(email + ":" + token)` derived from an `ApiToken`, and rate-limited (120 req/min per user). The only residual work is the manual device-test matrix, tracked under `docs/releases/maintenance.md`.

Target clients: NetNewsWire (macOS/iOS), Reeder (Fever mode), Readably.

---

### Telegram Inline Buttons (M — 1–3 days)

Add "Mark as read" and "Open in FeedFerret" reply buttons to Telegram keyword alert messages via the Bot API inline keyboard.

- [ ] Switch bot from send-only polling mode to webhook mode (required to receive button press callbacks)
- [ ] `POST /api/telegram/webhook` — receive `callback_query` events
- [ ] Verify `callback_query.data` signature to prevent spoofing
- [ ] On "Mark as read": call existing mark-read server action; respond with `answerCallbackQuery`
- [ ] On "Open": respond with `answerCallbackQuery` + `url` pointing to the article
- [ ] Update `docs/webhooks.md` with webhook setup instructions

---

### Nested Categories / Tier-2 Categories (L — 1–2 weeks)

The DB schema already supports `parentId` on `Category`. Work needed is purely UI and logic.

- [ ] Sidebar: tree rendering for 2-level nesting (expand/collapse parent categories)
- [ ] Sidebar: unread count aggregated across parent + all children
- [ ] Feed Management → Categories: parent picker when creating/editing a category
- [ ] Setting inheritance: `hideFromAllFeeds`, `spoilerMode` cascade parent → child
- [ ] OPML export: nested `<outline>` for sub-categories
- [ ] OPML import: detect nested `<outline>` and map to parent/child categories
- [ ] Max depth: 2 levels (grandparent categories not supported — avoids sidebar complexity)
- [ ] MCP: `create_category` / `update_category` extended with optional `parentId`

---

### Full Zod Schemas for Server Actions (M — 1–2 days)

Replace ad-hoc input validation in server actions with explicit `z.object()` schemas at every action entry point. No behaviour change — pure defensive hardening.

- [ ] Audit all files in `app/actions/` for actions without a top-level Zod schema
- [ ] Add `z.object({...}).parse(input)` at the top of each action
- [ ] Ensure `ZodError` is caught and returns a user-facing error message (not a raw stack trace)
- [ ] `lib/validation.ts` — centralise reusable field schemas (`feedUrl`, `labelName`, `searchQuery`, etc.)
