## Core Reader

A self-hosted, multi-user RSS reader built on Next.js 16, React 19, TypeScript, Prisma, and PostgreSQL/SQLite.

- Feed types: RSS, Atom, JSON Feed, HTML autodiscovery, XML
- Article reader with multiple layout modes (list, magazine, reader view)
- Dark mode and dynamic accent/secondary color theming
- Progressive Web App: installable, offline fallback, app shortcuts, lock-screen media controls
- Mobile UX: bottom navigation, swipe gestures, safe-area handling, landscape support
- Keyboard shortcuts throughout
- Empty states for all views
- Full-text extraction with CSS/XPath selector editor (Scout Studio) — see [docs/scout-studio.md](../scout-studio.md)

## Power-User Automation

- **Auto-mark-as-read rules** — query-based rules with live preview; articles matching the rule are marked read on sync
- **Keyword alerts** — trigger on any search token; deliver via in-app, push, email, webhook, Telegram, Gotify, or ntfy
- **Outbound webhooks** — HMAC-SHA-256 signed, configurable retry logic — see [docs/webhooks.md](../webhooks.md)
- **Saved searches** — shareable public page + RSS export for any search query
- Retention policies per feed
- Duplicate detection via SHA-256 URL normalization
- Feed authentication and custom fetch options (headers, cookies)

## Notifications

Five parallel delivery channels, all configurable per-user:

| Channel | Notes |
|---|---|
| Browser push | Web Push / VAPID, requires HTTPS |
| Email | SMTP, Resend, Postmark, Mailgun, SendGrid; configurable digest frequency |
| Telegram | Bot API, MarkdownV2 formatting |
| Gotify | `/message?token=` endpoint |
| ntfy | Topic URL, optional Bearer token |

See [docs/self-hosting-auth-email.md](../self-hosting-auth-email.md) for provider configuration.

## Google Reader API

Full compatibility layer for native RSS clients — see [docs/google-reader-api.md](../google-reader-api.md).

Verified client support: **Reeder** (macOS/iOS), **NetNewsWire**, **FeedMe** (Android), **ReadKit**

Implemented endpoints: stream listing, item contents (batch), mark-read with microsecond cutoff, unread counts, user-info, starred stream, oldest-first sort, `ot` timestamp filter.

Per-article AI summaries. Bring your own API key — no key is stored server-side beyond your own settings:

## Security Hardening

All shipped in v1.0.0 — see [docs/security.md](../security.md) for detail:

- Rate limiting on all API surfaces (sliding window, per-user and per-IP)
- Security headers: CSP, X-Frame-Options, X-Content-Type-Options, Referrer-Policy, Permissions-Policy
- API token hashing (SHA-256 stored, `ff_`-prefix format, one-time display)
- SSRF protection on all outbound feed-fetch URLs
- Input validation: URL 2 KB, OPML 5 MB, label 100 chars, search 1 000 chars
- Admin hardening: audit log, 2FA enforcement, session invalidation, login logging
- CVE patches via `pnpm.overrides`
