# v1.2 — Theming & Layout

> Status: Planned
> Estimated effort: L (large — 2–4 weeks)
> Depends on: v1.1 (no hard dependency, but i18n strings for theme UI should already be in `en.json`)

---

## Goal & Scope

Deep visual customization: theme presets, full per-zone color control, reader typography, and layout density modes. The goal is to make FeedFerret feel like the user's own without requiring CSS knowledge. Every setting lives in the database and roams with the user's account. This release also bundles the deferred accessibility items from the v1.0.0 sprint (A4–A5) and adds `axe-playwright` to CI.

---

## Milestone Checklist

### Theme System Architecture

- [ ] Audit all existing CSS variables and Tailwind utility classes — produce a complete token inventory
- [ ] Define `FeedFerretTheme` TypeScript interface in `lib/themes.ts`: all semantic token names, types, and default values
- [ ] Establish semantic token layer: `--sidebar-bg`, `--sidebar-fg`, `--reader-bg`, `--reader-fg`, `--article-list-bg`, `--header-bg`, `--accent`, `--accent-fg`, and the full set
- [ ] Extend `theme-color-applier.tsx` (or equivalent) to write all theme tokens to `:root` as CSS custom properties on mount and on change
- [ ] `User.themeJson` field added to Prisma schema (JSON column, nullable)
- [ ] Migration written and tested on both PostgreSQL and SQLite
- [ ] Backwards-compatible migration: existing `accentColor` and `secondaryColor` user fields are read on first load and converted to the new token format; legacy columns kept until v1.3 then dropped

### Built-in Presets

- [ ] Define the following presets in `lib/themes.ts`, each as a `FeedFerretTheme` object:
  - `oled-black` — pure black backgrounds for OLED displays
  - `solarized-light` — Solarized Light palette
  - `solarized-dark` — Solarized Dark palette
  - `catppuccin-mocha` — Catppuccin Mocha (dark)
  - `catppuccin-latte` — Catppuccin Latte (light)
  - `gruvbox-dark` — Gruvbox Dark
  - `high-contrast-dark` — high-contrast dark for accessibility
- [ ] Gallery picker in Settings → Appearance: visual swatches, live preview on hover, one-click apply
- [ ] Default (system light/dark) remains the out-of-the-box experience; presets are opt-in

### Advanced Color Customization

- [ ] Per-zone color pickers: Sidebar, Article List, Reader, Header — each independently configurable
- [ ] HSL sliders (Hue, Saturation, Lightness) with real-time preview; hex input accepted
- [ ] Live preview panel reflects changes without saving
- [ ] WCAG contrast checker: warn inline (yellow badge) when foreground/background contrast ratio falls below 4.5:1 (AA) for normal text or 3:1 for large text
- [ ] "Reset zone to preset" button per zone

### Reader Typography

- [ ] Font family picker with these options:
  - System UI (default)
  - Serif (Georgia / `font-serif`)
  - Monospace
  - OpenDyslexic (self-hosted WOFF2, loaded on demand)
  - Atkinson Hyperlegible (self-hosted WOFF2, loaded on demand)
- [ ] Font size slider: 14 px – 24 px, step 1 px
- [ ] Line height slider: 1.2 – 2.0, step 0.1
- [ ] Max measure (line length) slider: 50ch – 90ch; default 70ch
- [ ] Letter spacing: -0.05em – 0.1em slider
- [ ] Text alignment: Left / Justify (no Center/Right — avoid readability traps)
- [ ] All typography settings applied via CSS custom properties on the `.reader` container, not inline styles

### Layout Density

- [ ] Three density modes for the article list: **Comfortable** (default) / **Compact** / **Cozy**
- [ ] Each density adjusts: row height, image thumbnail size, line count, font size in list
- [ ] Stored in `User.themeJson` under `density` key

### Article List Layout Modes

- [ ] **List** — current default: title + source + date in a single row
- [ ] **Compact** — tighter rows, no thumbnails, maximum information density
- [ ] **Magazine** — large featured image, 2-column grid, excerpt shown
- [ ] **Cards** — card grid with image, title, excerpt; responsive column count
- [ ] Layout mode switcher in article list header (icon toggle group)
- [ ] Per-feed or per-category layout override (optional; store in feed/category metadata)

### Focus / Zen Mode

- [ ] Keyboard shortcut `z` toggles Zen mode in the article reader
- [ ] Zen mode: hides sidebar, article list, and all chrome — only article content visible
- [ ] `Escape` or `z` exits Zen mode
- [ ] Zen mode state is session-only (not persisted)

### Category Color Coding

- [ ] Optional per-category color dot in the sidebar (defaults to accent color)
- [ ] Color picker on category edit dialog
- [ ] Color stored in `Category` table as a nullable hex/HSL string
- [ ] Category color used as left-border accent in article list when articles are in that category

### Theme Export / Import

- [ ] "Export theme" button in Settings → Appearance: downloads `feedferret-theme.json` (the `FeedFerretTheme` object)
- [ ] "Import theme" button: accepts the same JSON, validates against the `FeedFerretTheme` schema before applying
- [ ] Shareable URL param: `?theme=<base64url(JSON)>` — visiting the URL auto-previews the theme with a "Save" prompt; does not apply without confirmation

### Admin Default Theme

- [ ] Server Management: "Default theme" dropdown — sets the theme applied to new users and unauthenticated pages
- [ ] Admin can also set default density and default article list layout mode

---

## Out of Scope for v1.2

- Custom CSS injection (arbitrary user CSS) — security surface; deferred indefinitely
- Per-article background color (reader view only; full-page theming is the scope)
- Font hosting CDN (all custom fonts are self-hosted WOFF2 in `public/fonts/`)
- RTL-aware theme tokens (handled in v1.1)
- Paid/premium themes

---

## Estimated Effort

**L — Large (2–4 weeks)**

Theme system architecture and the token migration are the highest-risk items. Presets and typography are mechanical once the token layer is in place. Accessibility work can proceed in parallel.
