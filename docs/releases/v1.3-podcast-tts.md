# v1.3 — Podcast & Text-to-Speech

> Status: Planned
> Estimated effort: XL (very large — ship as multiple sub-PRs)
> Depends on: v1.2 (no hard code dependency, but the audio player UI should use the finalized theme tokens)

---

## Goal & Scope

Turn FeedFerret into a full audio reading experience. Podcast feeds are subscribed to alongside RSS — episodes live in the same unified feed list, with a persistent audio player and queue. Any article can also be read aloud via Text-to-Speech: Web Speech API for instant zero-config use, Kokoro-82M on-device model for offline high-quality TTS, or the user's OpenAI API key for cloud-quality voice. This release is the largest in the v1.x cycle and must be split into independently mergeable sub-PRs.

## Milestone Checklist

### Podcast Feed Parsing

- [ ] Detect feed type as `PODCAST` when any enclosure or iTunes namespace tag is present; store on `Feed.feedType` enum (`RSS` | `ATOM` | `JSON` | `PODCAST`)
- [ ] Parse `<enclosure>` tags: `url`, `type`, `length`
- [ ] Parse iTunes namespace (`<itunes:*>`): `duration`, `image`, `explicit`, `episodeType`, `season`, `episode`, `summary`, `author`
- [ ] Parse Podcasting 2.0 namespace (`<podcast:*>`): `chapters` URL, `transcript` URL/type, `soundbite`, `person`, `location`, `value` (Lightning; store but do not activate)
- [ ] `Episode` DB model: `feedId`, `articleId` (optional link to article), `audioUrl`, `audioType`, `audioDuration`, `audioSize`, `itunesDuration`, `chaptersUrl`, `transcriptUrl`, `episodeNumber`, `seasonNumber`, `explicit`, `podcastImage`
- [ ] Episode records created and updated on feed sync; duration normalized to seconds
- [ ] OPML import/export extended to handle podcast feeds (standard `<outline type="rss">` — no special treatment needed)

### Audio Player

- [ ] Persistent mini-player docked to the bottom of the screen (above bottom navigation on mobile); always visible when an episode is loaded
- [ ] Full-screen player view (tap/click expand button on mini-player)
- [ ] Controls: play/pause, seek bar with timestamp, skip back 15 s, skip forward 15 s, playback speed (0.5× 0.75× 1× 1.25× 1.5× 1.75× 2×)
- [ ] Episode progress: persisted to `Episode.listenedSeconds` on pause and at regular intervals (every 30 s); resume from last position on re-open
- [ ] Background playback via [Media Session API](https://developer.mozilla.org/en-US/docs/Web/API/Media_Session_API): title, artist (podcast name), artwork, prev/next track actions
- [ ] PWA lock-screen controls wired to Media Session API
- [ ] Chapter support: if `chaptersUrl` is set, fetch chapters JSON and display chapter markers on seek bar; clicking a chapter marker seeks to it

### Podcast UX

- [ ] Podcast feeds get a dedicated "Podcast" section in the sidebar (below the main feed list or as a collapsible group)
- [ ] Unplayed episode counter badge on each podcast in the sidebar
- [ ] Episode queue: add episode to queue, reorder via drag-and-drop, auto-advance to next queued episode
- [ ] "Unplayed" filter in the podcast view (mirrors "Unread" for article feeds)
- [ ] Podcast-specific article card: shows duration, episode number, play button inline
- [ ] "Mark as played" action (equivalent of "mark as read")

### TTS — MVP (Web Speech API)

- [ ] Play button in the Article Reader toolbar opens TTS controls: play/pause/stop
- [ ] Uses browser [`SpeechSynthesis`](https://developer.mozilla.org/en-US/docs/Web/API/SpeechSynthesis) API — zero dependencies, works offline in supporting browsers
- [ ] Language detection: use `article.language` field if set; fall back to `navigator.language`
- [ ] Word highlight via `charIndex` from `SpeechSynthesisUtterance.onboundary` events — highlight the current word in the article text
- [ ] Speed control: 0.5× – 2×
- [ ] Resume: if the user navigates away and returns, offer to resume from the last character offset
- [ ] Graceful degradation: if `window.speechSynthesis` is undefined, TTS controls are hidden (not errored)

### TTS — On-Device (Kokoro-82M)

- [ ] Integrate [Kokoro-82M](https://huggingface.co/onnx-community/Kokoro-82M-ONNX) via `onnxruntime-web` or `@huggingface/transformers.js`
- [ ] Model loaded in a Web Worker to avoid blocking the main thread
- [ ] Service Worker caches the model files after first download; subsequent page loads use the cache
- [ ] Web Speech API used as automatic fallback if the model fails to load or the browser lacks WASM support
- [ ] Model download is opt-in: user must click "Download on-device voice model" in Settings → Audio (shows file size ~82 MB)
- [ ] Voice selection: the voices available in Kokoro-82M (American English male/female, British English)

### TTS — Cloud BYOK (OpenAI)

- [ ] Reuse existing AI provider credential mechanism (`lib/ai-providers.ts` or equivalent)
- [ ] Endpoint: OpenAI `POST /v1/audio/speech`, models `tts-1` (faster, lower quality) and `tts-1-hd` (slower, higher quality)
- [ ] Voice picker: alloy, echo, fable, onyx, nova, shimmer
- [ ] Audio streamed as MP3, played via the same audio element as the podcast player
- [ ] Cost warning displayed inline in settings: "Cloud TTS uses your OpenAI API credits. Approx. $0.015 per 1,000 characters (tts-1)."
- [ ] No server-side proxying of audio — the browser fetches directly from OpenAI using the user's key (or via a thin server-side streaming route if CORS is a problem)

### TTS Settings

- [ ] Settings → Audio section (new)
- [ ] Provider picker: Web Speech API (default) / On-Device (Kokoro) / OpenAI Cloud
- [ ] Global default voice selection (per provider where applicable)
- [ ] Global default speed
- [ ] On-device model management: download, storage usage, delete cached model

## Morning Briefing — Evaluation Gate (2.3)

Morning Briefing (a daily audio digest of top articles, auto-played on app open) is a v2.x feature. It will be moved to the active roadmap if:

1. TTS (any provider) ships stably in v1.3 with positive user feedback
2. The on-device Kokoro model demonstrates acceptable latency for article-length text on mid-range hardware

If the gate is met, Morning Briefing becomes the first item on the v2.0 scope. If not, it stays in the backlog.

---

## Out of Scope for v1.3

- Android/iOS native podcast apps or system-level podcast sync (e.g., gpodder.net API)
- Transcript display and search (data model supports it; UI deferred)
- Lightning/Value4Value payments (data model stores the `<podcast:value>` tag; no activation)
- Speech-to-text / voice search
- ElevenLabs or other TTS providers beyond OpenAI (can be added via the BYOK provider mechanism without a major release)
- Podcast recommendation engine
